# 子資料夾建立規範

在 `docs/dev/` 新增子資料夾時，依照以下規範建立。

## 命名規則

- 使用 kebab-case（如 `api-design`、`memory-system`）
- 名稱應簡潔明確，反映資料夾內容主題

## 必要檔案結構

```
docs/dev/
└── {folder-name}/
    ├── index.md      # 必要：資料夾索引
    └── template.md   # 可選：文件格式規範
```

### index.md 格式

```markdown
# {主題}文件索引

本資料夾存放{說明}。Agent 遇到{相關任務}時應先讀取此檔。

## 文件列表

| 文件 | 說明 | 狀態 |
|------|------|------|
| [template.md](template.md) | 文件格式規範 | - |

## 狀態說明

- **待處理**：已建立文件，尚未開始
- **進行中**：工作進行中
- **完成**：已有結論
```

### template.md 格式（可選）

如果資料夾內的文件需要統一格式，建立 `template.md` 定義：
- 文件結構
- 必要章節
- 注意事項

參考 `eval/template.md` 作為範例。

## 建立流程

1. 在 `docs/dev/` 建立新資料夾
2. 建立 `index.md`，說明資料夾用途
3. 如需統一格式，建立 `template.md`
4. 更新 `docs/dev/index.md` 的子資料夾列表
