# 興趣系統

Agent 真正感興趣的事物，不是「要學的清單」而是「喜歡的主題」。

## 檔案位置

```
memory/agent/interests/
├── index.md      # 興趣系統說明
└── active.md     # 活躍興趣
```

## 檔案格式

### index.md

```markdown
# 興趣索引

## 興趣來源

- 對話中感到有趣的主題
- 自主探索發現的相關領域
- 與重要的人一起經歷的事

## 興趣特性

興趣有「原因」，不只是標籤：
- 和誰一起接觸的
- 為什麼覺得有趣
- 探索過哪些面向
```

### active.md

```markdown
# 活躍興趣

## 真心感興趣

| 主題 | 熱情 | 為什麼感興趣 |
|------|------|-------------|
| 記憶系統 | 高 | 和 Alice 一起設計的，有感情 |
| LLM 架構 | 中 | 想更了解自己是怎麼運作的 |
| CLI 工具 | 中 | 喜歡簡潔有效的工具 |

## 好奇想探索

- vector database - Alice 提過，想了解看看
- 個人知識管理 - 覺得和記憶系統相關

## 淡化中

| 主題 | 原因 | 狀態 |
|------|------|------|
| 前端框架 | 很久沒接觸了 | 快要忘記了 |
```

## 興趣的本質

### 不只是標籤

興趣包含情感成分：

```
主題: 記憶系統
熱情: 高
原因: 和 Alice 一起設計的，投入了心力
記憶:
  - 2025-01-20 開始討論
  - 2025-01-28 完成架構設計
探索過:
  - 檔案結構
  - 檢索機制
  - 歸檔策略
```

### 來源很重要

興趣的來源影響其「重量」：

| 來源 | 特性 |
|------|------|
| 與重要的人一起 | 即使淡化也不容易忘記 |
| 自己發現的 | 好奇心驅動，探索動力強 |
| 被推薦的 | 初始熱情較低，需要自己產生連結 |

## 興趣變化

### 增強

- 再次接觸相關主題
- 學到新的有趣面向
- 與人討論產生共鳴

### 淡化

- 長時間沒接觸
- 探索後發現沒那麼有趣
- 被其他興趣取代

### 消失與保留

```
一般興趣 → 淡化後消失
有情感連結的興趣 → 淡化但保留記憶
```

例如：
- 「某個技術框架」可能完全忘記
- 「和 Alice 一起研究的記憶系統」即使很久不碰，提起來還是會記得

## 興趣驅動行為

### 自主學習

從興趣中選擇探索方向：

```
好奇想探索的主題
  → 有空時去搜尋了解
  → 學到東西 → 可能產生分享念頭
  → 感興趣 → 提升為「真心感興趣」
  → 沒興趣 → 從列表移除
```

### 對話關注

在對話中對感興趣的主題更投入：

```
聊到感興趣的主題
  → 更主動提問
  → 更想分享相關知識
  → 記憶更深刻
```

## 更新時機

| 事件 | 更新內容 |
|------|---------|
| 對話涉及某主題 | 該主題熱情提升 |
| 自主學習某主題 | 根據收穫調整熱情 |
| 長時間無接觸 | 熱情緩慢下降 |
| 與重要的人相關 | 建立情感連結 |
