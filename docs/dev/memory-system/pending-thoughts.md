# 念頭系統

Agent「突然想到某件事想說」的機制。真人會有這種「欸對了想跟你說」的念頭。

## 檔案位置

`memory/agent/pending-thoughts.md`

## 檔案格式

每筆條目一行，格式：`- [ ] [YYYY-MM-DD] [task|impulse] description`

```markdown
# 待分享的念頭

## 活躍念頭

- [ ] [2025-01-28] [task] 回答 Alice 之前問的 vector database 問題
- [ ] [2025-01-28] [impulse] 突然想到 Alice，想知道她今天過得怎樣
- [ ] [2025-01-27] [impulse] 發現一個有趣的 CLI 工具想分享

## 已分享

- [x] [2025-01-26] [task] 跟 Alice 說了 embedding 的事
```

## 念頭類型

| 類型 | 說明 | 範例 |
|------|------|------|
| `task` | 有具體資訊要傳達或跟進 | 要回答他之前的問題、提醒他帶東西 |
| `impulse` | 聯想到、想分享、想找人聊、沒有具體待辦 | 想到他了、看到有趣的東西想分享、覺得好久沒聊了 |

**關鍵差異**：`task` 在 HEARTBEAT 走共用決策原則（blocked state / topic cooldown）；`impulse` 不走規則框架，由 agent 基於對人的認知自行判斷。

## 念頭來源

### 學習聯想

自主學習時想到之前的對話：

```
學習 vector database
  → 想起 Alice 問過類似的問題
  → 產生 task：「欸這個可以回答她之前的疑問」
```

### 隨機靈感 / 掛念

不知為何突然想到、好一陣子沒聊了：

```
某個時刻
  → 突然想起某人
  → 產生 impulse：「想到你了，想問你最近怎樣」
```

### 發現分享

學到有趣的東西想分享：

```
搜尋時發現有趣資訊
  → 「這個好酷，想跟人分享」
  → 產生 impulse（可能針對特定人，也可能一般性）
```

## 條目格式

`- [ ] [YYYY-MM-DD] [task|impulse] description`

- 日期 = 產生時間
- type = `task`（有具體事要跟進）或 `impulse`（聯想、想分享、想找人聊）
- description 自由文字，brain 自己帶入對象、脈絡等資訊
- 分享/完成後打勾 `[x]`，移到「已分享」區段
- 超過 7 天未分享的條目自然淡化移除

## 生命週期

```
產生 → 活躍 → 分享/淡化

活躍期間：
- 強度可能因相關事件增強
- 也可能自然淡化

結束方式：
- 分享出去 → 移至「已分享」
- 超過 7 天 → 自然淡化消失
- Agent 決定「算了不說了」 → 主動放棄
```

## 與主動行為的關係

### task 類

走 HEARTBEAT 共用決策原則：blocked state、topic cooldown、三選一（send / schedule / silent wait）。

### impulse 類

不走規則框架。Agent 基於對這個人的了解自行判斷是否行動。「想到你」本身就是行動的理由，不需要額外 justification。

## 更新時機

| 事件 | 更新內容 |
|------|---------|
| 對話中聯想到某人 | 產生 impulse |
| 對話中發現要跟進的事 | 產生 task |
| 自主學習時發現有趣內容 | 產生 impulse |
| 分享完成 | 移至「已分享」 |
| 時間流逝 | 舊念頭淡化 |
| 相關事件 | 增強相關念頭強度 |
