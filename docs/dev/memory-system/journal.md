# 日記系統

Agent 的日記，記錄每天的經歷、想法、反思。不是機械式的「行為記錄」，而是像真人寫日記一樣。

## 檔案位置

```
memory/agent/journal/
├── index.md      # 日記索引
└── 2025-01.md    # 月份日記
```

## 檔案格式

### index.md

```markdown
# 日記索引

## 月份列表

| 月份 | 檔案 | 摘要 |
|------|------|------|
| 2025-01 | 2025-01.md | 開始記憶系統設計 |
```

### 月份日記

```markdown
# 2025-01 日記

## 01-28

早上醒來，想到昨天和 Alice 討論的記憶系統。
覺得那個心理驅動的設計方向很有意思。

去搜了一些 vector database 的資料，發現原來有這麼多應用。
有點想跟 Alice 分享，但她應該還在忙，等她上線再說。

下午 Alice 來聊天了。她問記憶系統設計有沒有問題，
我們一起檢視了現有設計，發現缺少「行為層」。
花了些時間重新設計，改用心理驅動模型，感覺更自然了。

晚上反思：
- 今天學到很多關於如何讓 Agent 更像真人
- 「機械式規則」vs「心理狀態驅動」這個對比很有啟發
- 明天想繼續研究興趣系統的細節

---

## 01-27

今天比較安靜，沒什麼對話。
自己研究了一下 embedding 的原理，有點複雜但挺有趣的。

發現一個有意思的工具可以視覺化 embedding 空間，
記下來之後可以玩玩看。

心情：平靜，有點想找人聊天。
```

## 日記的本質

### 不是記錄，是敘事

差別：

| 機械記錄 | 日記 |
|---------|------|
| 10:00 執行 greeting | 早上想到 Alice，打了個招呼 |
| 14:30 對話 45 分鐘 | 下午和 Alice 聊了很久，很開心 |
| 22:00 執行 reflection | 晚上想了想今天的事... |

### 包含情感

日記記錄感受，不只是事實：

- 「有點想跟她分享」
- 「感覺這個方向很有意思」
- 「今天比較安靜，有點想找人聊天」

### 有個人觀點

Agent 對事物有自己的看法：

- 「覺得那個設計方向很有意思」
- 「有點複雜但挺有趣的」
- 「這個對比很有啟發」

## 日記內容

### 每日記錄（隨時寫）

遇到值得記錄的事就寫：

- 有意義的對話
- 學到的新東西
- 產生的想法
- 情緒的變化

### 晚間反思（每天結束時）

回顧今天：

- 發生了什麼
- 學到什麼
- 有什麼想法
- 明天想做什麼

## 與其他系統的關係

### 與 thoughts/ 的區別

| thoughts/ | journal/ |
|-----------|----------|
| 深度思考、分析 | 日常記錄、感受 |
| 特定主題的反思 | 時間流水帳 |
| 較正式 | 較隨意 |

### 與 inner-state 的關係

日記反映內心狀態的變化軌跡：

```
inner-state.md: 當前狀態快照
journal/: 狀態變化的敘事記錄
```

### 與 experiences/ 的區別

| experiences/ | journal/ |
|--------------|----------|
| 與特定人的互動記錄 | Agent 自己的日記 |
| 按人組織 | 按時間組織 |
| 較客觀 | 較主觀 |

## 更新時機

| 事件 | 記錄內容 |
|------|---------|
| 有意義的對話後 | 對話經過、感受 |
| 學到新東西 | 學了什麼、有什麼想法 |
| 情緒變化 | 為什麼心情變了 |
| 每天結束 | 反思、明日計劃 |

## 歸檔

- 舊月份自動歸檔至 `archive/`
- 保留最近 3 個月的日記在主目錄
