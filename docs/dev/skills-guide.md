# Skills 指南

在 `docs/dev/` 建立或修改文件時，**必須先讀取本指南**以確保統一規範。

## 何時需要子資料夾

| 情境 | 結構 |
|------|------|
| 單一主題，內容獨立 | 直接建立 `.md` 檔 |
| 多個相關文件、需要 template | 建立子資料夾 |

**原則**：簡單優先，只在必要時建立子資料夾。

## 建立單一文件

直接在 `docs/dev/` 建立 `.md` 檔，完成後更新 `docs/dev/index.md` 的文件列表。

## 建立子資料夾

### 命名規則

- 使用 kebab-case（如 `api-design`、`memory-system`）
- 名稱應簡潔明確，反映資料夾內容主題

### 必要結構

```
docs/dev/
└── {folder-name}/
    ├── index.md      # 必要：資料夾索引
    └── template.md   # 可選：文件格式規範
```

### index.md 格式

```markdown
# {主題}文件索引

本資料夾存放{說明}。Agent 遇到{相關任務}時應先讀取此檔。

## 文件列表

| 文件 | 說明 | 狀態 |
|------|------|------|
| [template.md](template.md) | 文件格式規範 | - |

## 狀態說明

- **待處理**：已建立文件，尚未開始
- **進行中**：工作進行中
- **完成**：已有結論
```

### template.md（可選）

如果資料夾內的文件需要統一格式，建立 `template.md` 定義：
- 文件結構
- 必要章節
- 注意事項

參考 `eval/template.md` 作為範例。

## 建立流程

1. 決定是否需要子資料夾（參考上表）
2. 建立檔案或資料夾
3. 如為子資料夾，建立 `index.md`
4. 更新 `docs/dev/index.md`

## 現有範例

| 資料夾 | 用途 |
|--------|------|
| `eval/` | 技術評估與測試文件 |
| `task/` | 待辦任務管理 |
| `project-setup/` | 專案環境設置 |
