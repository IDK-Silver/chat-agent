# 測試文件格式規範

建立新的評估/測試文件時，依照以下格式：

---

```markdown
# [技術/工具名稱]

## 目的

為什麼要評估這個？要解決什麼問題？

## 評估重點

- 重點 1
- 重點 2
- ...

## 狀態

待測試 | 進行中 | 完成

## 測試紀錄

（測試過程中的發現、問題、數據）

## 結論

（最終決定：採用/不採用/待定，以及原因）
```

---

## 注意事項

- 檔名使用 kebab-case，如 `ollama-cloud.md`
- 建立新文件後，更新 `index.md` 的文件列表
- 結論應簡潔明確，方便日後查閱
