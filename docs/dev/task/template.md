# 任務文件模板

建立任務文件時，依照以下格式撰寫。

## 格式

```markdown
# {任務名稱}

{一句話描述目標}

## 背景

{為什麼需要這個任務？解決什麼問題？}

## 步驟

1. {步驟一}
2. {步驟二}
3. ...

## 驗證

- {如何確認步驟正確完成}
- {測試方式或檢查點}

## 完成條件

- [ ] {條件一}
- [ ] {條件二}
- [ ] ...
```

## 撰寫原則

- **目標**：一句話，明確說明任務要達成什麼
- **背景**：簡短說明動機，不需長篇大論
- **步驟**：具體可執行，避免模糊描述
- **驗證**：讓執行者知道如何確認做對了
- **完成條件**：checkbox 格式，方便追蹤進度

## 範例

```markdown
# 建立 API 認證模組

實作 JWT 認證機制供 API 端點使用。

## 背景

目前 API 端點無認證保護，需要加入 JWT 驗證確保安全性。

## 步驟

1. 建立 `src/auth/jwt.py` 模組
2. 實作 token 產生與驗證函式
3. 建立 FastAPI dependency 供路由使用
4. 加入單元測試

## 驗證

- `uv run pytest tests/auth/` 測試通過
- 手動測試：無 token 請求應回傳 401

## 完成條件

- [ ] jwt.py 模組建立完成
- [ ] 測試覆蓋率 > 80%
- [ ] 文件更新
```
