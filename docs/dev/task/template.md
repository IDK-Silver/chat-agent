# 任務文件模板

建立任務文件時，依照以下格式撰寫。根據任務複雜度選擇適合的模板。

## 簡單任務模板

適用於：bug fix、小改動、單一檔案修改

```markdown
# {任務名稱}

{一句話描述目標}

## 背景

{為什麼需要這個任務？解決什麼問題？}

## 步驟

1. {步驟一}
2. {步驟二}
3. ...

## 驗證

- {如何確認步驟正確完成}
- {測試方式或檢查點}

## 完成條件

- [ ] {條件一}
- [ ] {條件二}
```

## 設計任務模板

適用於：新功能、架構設計、多檔案變更、需要技術決策的任務

```markdown
# {任務名稱}

{一句話描述目標}

## 背景

{為什麼需要這個任務？解決什麼問題？}

## 設計決策

### {決策主題一}

- **選擇**：{選了什麼}
- **原因**：{為什麼選這個}
- **替代方案**：{考慮過但沒選的方案，為什麼不選}

### {決策主題二}

...

## 檔案結構

```
project/
├── {新增或修改的檔案}
├── {說明每個檔案的職責}
└── ...
```

## 技術設計

### {模組/元件名稱}

{職責說明}

```python
# 介面定義或關鍵程式碼結構
class Foo:
    def bar(self, x: int) -> str: ...
```

### {另一個模組}

...

## 步驟

1. {步驟一}
2. {步驟二}
3. ...

## 驗證

- {如何確認步驟正確完成}
- {測試方式或檢查點}

## 完成條件

- [ ] {條件一}
- [ ] {條件二}
```

## 撰寫原則

### 核心原則：自足性

**任務文件必須讓其他 session 能獨立執行，不需要知道之前的討論內容。**

這意味著：
- 討論過的設計決策都要寫進去
- 選擇 A 而非 B 的原因要記錄
- 介面定義要明確，不能只寫「實作 XXX」
- 檔案結構要完整列出

### 各區塊撰寫要點

| 區塊 | 要點 |
|------|------|
| 背景 | 簡短說明動機，不需長篇大論 |
| 設計決策 | 記錄「為什麼」，不只是「是什麼」 |
| 檔案結構 | 完整目錄樹，標註每個檔案職責 |
| 技術設計 | 介面定義、資料結構、關鍵邏輯 |
| 步驟 | 具體可執行，避免模糊描述 |
| 驗證 | 讓執行者知道如何確認做對了 |
| 完成條件 | checkbox 格式，方便追蹤進度 |

## 範例

### 簡單任務範例

```markdown
# 修復登入頁面樣式錯位

登入按鈕在手機版顯示位置偏移，需修正 CSS。

## 背景

用戶回報手機版登入頁面按鈕跑版。

## 步驟

1. 開啟 `src/styles/login.css`
2. 修正 `.login-btn` 的 `margin` 值
3. 測試 375px 寬度下的顯示

## 驗證

- Chrome DevTools 模擬 iPhone SE 寬度，按鈕置中顯示

## 完成條件

- [ ] 手機版按鈕位置正確
```

### 設計任務範例

```markdown
# 建立快取模組

實作應用層快取，減少資料庫查詢。

## 背景

部分 API 端點回應慢，經分析是重複查詢資料庫。需要加入快取層。

## 設計決策

### 快取策略

- **選擇**：Write-through + TTL
- **原因**：資料一致性重要，允許短暫延遲
- **替代方案**：Write-back（效能更好但一致性風險高，不適合此場景）

### 儲存方式

- **選擇**：Redis
- **原因**：已有 Redis 基礎設施，支援 TTL，適合分散式
- **替代方案**：本地記憶體快取（無法跨 instance 共享）

## 檔案結構

```
src/
├── cache/
│   ├── __init__.py
│   ├── client.py      # Redis client 封裝
│   ├── decorator.py   # @cached decorator
│   └── keys.py        # Key 生成邏輯
└── ...
```

## 技術設計

### CacheClient

封裝 Redis 操作，提供統一介面。

```python
class CacheClient:
    def get(self, key: str) -> str | None: ...
    def set(self, key: str, value: str, ttl: int = 300): ...
    def delete(self, key: str): ...
```

### @cached decorator

```python
@cached(ttl=60, key_prefix="user")
def get_user(user_id: int) -> User:
    ...
```

## 步驟

1. 建立 `src/cache/` 目錄結構
2. 實作 `CacheClient`
3. 實作 `@cached` decorator
4. 加入單元測試
5. 套用到慢速 API 端點

## 驗證

- `uv run pytest tests/cache/` 通過
- 套用快取的 API 回應時間 < 50ms（原本 > 200ms）

## 完成條件

- [ ] CacheClient 實作完成
- [ ] @cached decorator 可用
- [ ] 測試覆蓋率 > 80%
- [ ] 至少一個 API 端點套用快取
```
