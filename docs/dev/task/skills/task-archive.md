# 歸檔任務流程

任務完成後，依照此流程歸檔。

## 觸發時機

- 任務的所有完成條件皆已勾選
- Agent 完成任務工作後，應讀取此文件執行歸檔流程

## 流程

### 1. 確認完成條件

檢查任務文件中的完成條件：
- 所有 checkbox 應為 `[x]`
- 若有未完成項目，先完成或與用戶討論

### 2. 更新任務狀態

在 [index.md](index.md) 將任務狀態改為「完成」：

```markdown
| [task-name.md](task-name.md) | 說明 | 完成 |
```

### 3. 詢問用戶是否歸檔

向用戶確認：

> 任務「{任務名稱}」已完成。是否要歸檔此任務？
> - 歸檔：移動至 archive/ 資料夾
> - 保留：維持在主列表中

### 4. 執行歸檔（若用戶同意）

#### 4.1 移動任務文件

```
docs/dev/task/{task-name}.md → docs/dev/task/archive/{task-name}.md
```

#### 4.2 加入歸檔日期

在任務文件開頭加入：

```markdown
> **歸檔日期**：YYYY-MM-DD
```

#### 4.3 更新索引

- 從 [index.md](index.md) 任務列表移除
- 加入 [archive/index.md](archive/index.md)

### 5. 回報用戶

告知用戶歸檔完成，或保留在主列表。

## 注意事項

- 歸檔是可選的，不強制執行
- 歸檔後的任務仍可在 archive/ 查閱
- 若需要重新處理已歸檔任務，可移回主目錄
