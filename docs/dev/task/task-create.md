# 建立任務流程

當需要建立新任務時，依照此流程執行。

## 觸發時機

- 用戶明確要求建立任務
- 工作範圍較大，需要拆分追蹤

## 流程

### 1. 確認任務資訊

向用戶確認：
- **任務名稱**：簡潔描述（用於檔名，kebab-case）
- **目標**：一句話說明要達成什麼
- **背景**：為什麼需要這個任務

### 2. 建立任務文件

依 [template.md](template.md) 格式建立文件：

```
docs/dev/task/{task-name}.md
```

檔名規則：
- 使用 kebab-case
- 簡短但能辨識內容
- 例：`api-auth.md`、`refactor-db-layer.md`

### 3. 更新任務索引

在 [index.md](index.md) 的任務列表加入新任務：

```markdown
| [{task-name}.md]({task-name}.md) | {一句話說明} | 待處理 |
```

### 4. 回報用戶

告知用戶：
- 任務文件已建立
- 文件路徑
- 下一步動作（若有）

## 注意事項

- 任務應該是可完成的單位，避免過大
- 若任務太大，建議拆成多個子任務
- 建立後狀態預設為「待處理」
